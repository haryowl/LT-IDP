TERMS AND PROCEDURES FOR CONTINUOUS WASTE WATER MONITORING DATA SENDING (SPARING) IN 2025

A.	INTRODUCTION
1.	URL :  sparing.kemenlh.go.id 
2.	Monitoring Data Delivery is per hour, data with an interval of 2 minutes, so the total data sent is 30 (thirty) data, in one data delivery package
3.	The data sent at the Nth hour is the data at the N-1st hour, for example, at 09.00, it will send data at 08.00.00, 08.02.00, 08.04.00, 08.06.00, …, 08.58.00

ii.	METHOD : GET

C.	ID LOGGER  
1.	LOGGER ID is a unique code for each LOGGER owned by the company and has been registered by KLH/BPLH
2.	LOGGER ID is used as UID (Unique Identification) when sending data

D.	DATA SENDING FORMAT

1.	DATA VARIABLES:
i.	uid: STRING data type in the form of a LOGGER ID;
ii.	datetime: INTEGER data type in the form of a time in UNIX TIMESTAMP format
iii.	pH: NUMBER data type in the form of a pH concentration value
iv.	cod: NUMBER data type in the form of a COD concentration value (mg/l);
v.	tss: NUMBER data type in the form of a TSS concentration value (mg/l);
vi.	nh3n: NUMBER data type in the form of a NH3N concentration value (mg/l);
vii.	debit: NUMBER data type in the form of a DEBIT concentration value (m3/minute);

2.	DATA TYPE:
i.	STRING is a data type consisting of numbers and/or alphabets.
ii.	INTEGER is a data type consisting of whole numbers;
iii.	NUMBER is a data type consisting of whole numbers or decimal numbers;
iv.	UNIX TIMESTAMP is an INTEGER data type that represents time;

3.	DATA FORMAT:
i.	Data format is in JSON (JavaScript Object Notation);
ii.	JSON is a popular data format used in system/application integration;

E.	DATA ENCRYPTION

1.	Encryption using JWT (JSON Web Tokens), a reference can be found at https://jwt.io/;
2.	JWT Header;  
{  
 	"typ": "JWT",  
 	"alg": "HS256" 
}  

3.	Encrypt the HEADER and DATA using Base64UrlEncode and HMACSHA256;
4.	Combine the results of the HEADER encryption and DATA encryption with the dot (.) character;

dataEncode = Base64UrlEncode(HEADER) + "." + Base64UrlEncode(DATA); 

5.	Create SECRET KEY from API KEY & DATA ENCODE using HMACSHA256;
6.	KEY SECRET encryption using Base64UrlEncode;

keySecret = Base64UrlEncode(HMACSHA256(dataEncode, API_KEY)); 


7.	Combine dataEncode and keySecret with a Dot (.) Character;

dataSend = dataEncode + "." + keySecret;

function base64url_encode($data){  
$b64 = base64_encode($data);  
 	if ($b64 === false) return false;  
 	$url = strtr($b64, '+/', '-_');  
 	return rtrim($url, '=');  
}

$dataConversion = base64url_encode($jsonHeader) . '.' .  base64url_encode($jsonData2Minute);
